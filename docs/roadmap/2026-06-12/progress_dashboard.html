HIGH qual.py가 fallback 없이 rag/retriever를 직접 호출

DB·임베딩 모델이 없는 환경에서 파이프라인 전체가 크래시합니다. 현재 CI에서 관련 테스트 3건을 임시 제외한 상태로, 안전망이 이 경로를 커버하지 못하고 있습니다.

  • 조치 필요: Qual 담당자가 retriever 호출에 try/except + mock fallback 복원
  • fallback 복원 후 CI 제외 테스트 3건 재활성화
  • 루브릭 #2(에러핸들링&폴백)가 4점으로 확정되려면 이 회귀 해소가 선행 조건

공지 [영역 외 수정] Curator corp_code 누락 보정 — PM(이동원)

RAGAS 실측 과정에서 Curator가 삼성전자 외 모든 종목에 corp_code를 반환하지 않는 버그를 발견했습니다. 에이전트 설계 원칙("KRX 종목코드와 DART corp_code를 함께 반환")에 어긋나며, DB 연결 환경에서 Quant의 DART 재무 조회가 전 종목 실패해 E2E와 RAGAS 측정이 차단되는 상태였습니다 (DB 없던 시절엔 quant fallback에 가려져 보이지 않던 버그).

  • 수정 범위 (로직 변경 없음): ① 카탈로그 corp_code 채움 — DART 공식 API 검증값 (SK하이닉스 00164779, KB금융 00688996, 신한지주 00382199) ② corp_code가 비면 stock_code로 카탈로그에서 보충하는 _resolve_corp_code 헬퍼 추가
  • 검증: 기존 테스트 45개 전부 통과 + E2E 5케이스 정상 동작 확인
  • Curator 담당자께: PR 리뷰 부탁드립니다. 다른 방식(예: DB lookup)을 원하시면 되돌리기 쉽게 별도 커밋으로 분리해 두었습니다 🙏
